# `<head>` vs `<header>`
## The `<head>` Tag (Invisible Metadata)
- **Purpose**: Contains machine-readable information (metadata) about the document, such as the page title, links to stylesheets, and SEO meta tags.
- **Placement**: It sits at the very top of your HTML file, right after `<html>` and before `<body>`.
- **Visibility**: Its contents are **never** displayed on the webpage itself (with the exception of the `<title>` appearing in the browser tab).
- **Reference**: Learn more via the MDN.

## The `<header>` Tag (Visible Content)
- **Purpose**: Represents a container for introductory content or navigational links for a specific section of the page. It often holds site logos, search forms, or top menus.
- **Placement**: It sits inside the `<body>` tag. You can use multiple `<header>` tags on a single page (e.g., at the top of an `<article>` or `<section>`).
- **Visibility**: Its contents are **visible** to the user.
- **Reference**: Learn more via the MDN `<header>` Reference.

## The Big Picture
Think of the `<head>` as the "behind-the-scenes" file manager, and the `<header>` as the visible header/banner at the top of the actual website.
